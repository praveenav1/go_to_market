# Admin Quick Reference - GTM Submission Workflow

## How to Access Admin Panel

1. **Open GTM Repository** at your application URL
2. **Press the 'A' key** anywhere on the page to reveal the admin controls
3. Click the **"🔐 Admin Panel"** button that appears in the header
4. You're now in the Admin Dashboard

## Admin Dashboard Overview

### Status Filters
At the top of the Admin Panel, you'll see four buttons:
- **Pending**: New submissions waiting for review (default view)
- **Approved**: Already approved resources
- **Rejected**: Rejected submissions
- **All**: Show all submissions

### Reviewing a Submission

1. **Find the submission** you want to review in the list
2. Click the **"Review"** button (only visible for pending submissions)
3. A modal dialog will appear with options to:
   - **Add Review Notes** (optional) - Document why you're approving/rejecting
   - **Approve** - Add the resource to the GTM repository
   - **Reject** - Decline the submission
   - **Cancel** - Close without taking action

### Understanding Submission Details

Each submission card displays:
- **Resource Title** (Header)
- **Description** of the GTM resource
- **Submitted Date/Time** - When user submitted it
- **Review Date/Time** - When admin reviewed it (if already reviewed)
- **Tags** - Categorization tags for filtering
- **Status Badge** - Current status (Pending/Approved/Rejected)
- **Admin Notes** - Any notes from previous admin review (if applicable)

## Approval Process

### When to Approve
✅ Resource content is accurate  
✅ Video quality is acceptable  
✅ Description is clear and helpful  
✅ Tags are appropriate  
✅ Resource adds value to GTM repository  

### When to Reject
❌ Resource content is incomplete or inaccurate  
❌ Video is corrupted or poor quality  
❌ Description is unclear or inappropriate  
❌ Resource duplicates existing content  
❌ Resource doesn't fit GTM purpose  

## After Approval

- ✅ Resource **automatically appears** in the main GTM Repository
- ✅ All users can **search and filter** the new resource
- ✅ Resource has a **unique ID** assigned automatically

## After Rejection

- ❌ Resource does **NOT** appear in the repository
- ❌ Resource stays in "Rejected" section for your records
- ❌ User can resubmit with improvements

## Best Practices

1. **Always add review notes** when rejecting - helps user improve
2. **Check for duplicates** before approving
3. **Verify video quality** - Click to preview if needed
4. **Maintain consistency** in tag usage across approvals
5. **Review regularly** to keep pending queue clear

## Troubleshooting

### Admin Button Not Showing
- **Solution**: Press the 'A' key again on any page

### Submission Won't Load
- **Solution**: Refresh the page and try again
- **Contact IT** if problem persists

### Video Upload Failed
- **Reason**: May be file size or format issue
- **Request**: Ask user to resubmit with valid MP4 file

## Submission Status Reference

| Approved | Added to repository | ❌ No | Now live for all users |
| Rejected | Not added | ❌ No | Check rejection notes |
